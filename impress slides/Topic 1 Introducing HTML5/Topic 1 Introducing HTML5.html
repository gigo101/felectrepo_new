Topic: Introducing HTML5

HTML is the language you use to write web pages.

In 1998, the W3C stopped working on HTML and attempted 
to improve it with an XML-powered successor called 
XHTML 1.0.


XHTML 1.0: Getting Strict

example:
<h1>The Life of a <i>Duck</i></h1>

And you accidentally swap the final two tags:
<h1>The Life of a <i>Duck</h1></i>




When a browser encounters this slightly messed-up markup, 
it can figure out whatyou really want.  It italicizes the last word, without even a polite complaint.

However,the mismatched tags break the official rules of 
XHTML


-XHTML has most of the same syntax conventions as HTML, 
but it enforces stricter rules. 

XHTML 2: The Unexpected Failure

-It was set to tighten up the errorhandling rules, 
forcing browsers to reject invalid XHTML 2 pages

-XHTML 2 dumped a few well-worn elements:
<b> (for bold text), <i> (for italics), and <iframe> (for
embedding one web page inside another).

-Development on XHTML 2 dragged on for five years, 
and developer enthusiasm  slowly leaked away.

HTML5: Back From the Dead

-2004, a group of people started looking at the future of the 
Web from a different angle. 

-WHATWG (Web Hypertext Application
Technology Working Group)- think of new solutions.

-Standards evolve into HTM5
Three Key Principles of HTML5

1. Don’t Break the Web
     -standard shouldn’t introduce changes that make
other people’s web pages stop working

2. Pave the Cowpaths
     -HTML5 aims to standardize these unofficial (but widely 
       used) techniques.

3. Be Practical
     - Changes should have a practical purpose

The HTML5 Doctype

-The first line of every HTML5 document is a special code 
called doctype.

<!DOCTYPE html>

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//
EN""http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">


The Old XHTML 1 Doctype
Why does your web page require any doctype at all?

-Without a doctype, most browsers (including Internet Explorer 
and Firefox) will lapse into quirks mode.

-When you add a doctype, the browser recognizes that you want
 to use the stricter standards mode

-The HTML5 doctype is simply the shortest valid doctype, so it 
triggers standards mode.

Character Encoding

-The character encoding is the standard that tells a computer 
how to convert your text into a sequence of bytes when it’s 
stored in a file (and how to convert it back again when the file 
is opened)

-Today, virtually all English websites use an encoding called
UTF-8

<head>
<meta charset="utf-8">
<title>A Tiny HTML Document</title>
</head>

The Language

-It’s considered good style to indicate your web page’s natural 
language

-The easiest way to add language information to your web page 
is to use the <html>element with the lang attribute:
<html lang="en">

Adding a Style Sheet

-Virtually every web page in a properly designed, professional 
website uses style sheets. You specify the style sheets you want
to use by adding <link> elements to the <head> section of an 
HTML5 document, like this:

<head>
<meta charset="utf-8">
<title>A Tiny HTML Document</title>
<link href="styles.css" rel="stylesheet">
</head>

Adding JavaScript

-You add JavaScript to an HTML5 page in much the same way 
that you add it to a traditional HTML page.

<head>
<meta charset="utf-8">
<title>A Tiny HTML Document</title>
<script src="scripts.js"></script>
</head>

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
<title>A Tiny HTML Document</title>
<link href="styles.css" rel="stylesheet">
<script src="scripts.js"></script>
</head>
<body>
<p>Let's rock the browser, HTML5 style.</p>
</body>
</html>
